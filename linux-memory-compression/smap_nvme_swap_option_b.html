1. Design Overview & Rationale

Option B adds NVMe swap capability to SMAP entirely in user space, leveraging the kernel’s existing swap infrastructure. Instead of implementing custom block I/O and swap entry management, SMAP uses process_madvise(MADV_PAGEOUT) to request the kernel to reclaim specific cold pages, which are then written to a pre-configured swap device by the kernel’s normal swap path.

Key Advantage: Zero kernel module changes. The entire feature is implemented in SMAP user-space (libsmap.so). The kernel handles all swap I/O, PTE manipulation, swap entry allocation, and page fault handling transparently.

3. NVMe Swap Device Setup

3.1 Swap Device Configuration

SMAP manages the NVMe swap device lifecycle. The swap device can be a dedicated NVMe partition or a swap file on an NVMe filesystem. Partition is preferred for production; file is easier for testing.

struct SwapDeviceConfig {
    char device_path[256];     // e.g., "/dev/nvme0n1p3" or "/swap/swapfile"
    enum { SWAP_PARTITION, SWAP_FILE } type;
    uint64_t size_bytes;        // Swap device capacity
    int priority;               // swapon priority (0-32767, higher = preferred)
    bool enable_discard;        // TRIM/discard for NVMe (recommended)
    bool auto_swapon;           // Auto-enable on SMAP start
};

struct SwapPolicy {
    uint32_t cold_window_threshold;  // Consecutive zero-freq windows before swap (default: 5)
    uint64_t max_swap_per_cycle;     // Max pages to swap per ScanMigrateWork cycle
    uint64_t max_swap_per_process;   // Max swap bytes per process (0 = unlimited)
    double l2_watermark_ratio;       // Start swapping when L2 usage exceeds this ratio (e.g., 0.85)
    uint32_t batch_size;             // iovec entries per process_madvise call (default: 64)
    bool swap_enabled;               // Global swap enable/disable
};

3.2 Initialization & swapon

// SwapDeviceManager::Init()
int SwapDeviceInit(struct SwapDeviceConfig *cfg) {
    // 1. Validate device exists and is NVMe
    if (access(cfg->device_path, F_OK) != 0)
        return -ENODEV;

    // 2. Check if already swapon'd
    if (IsDeviceAlreadySwap(cfg->device_path))
        return 0;  // Already active

    // 3. Format as swap (only if SWAP_PARTITION and not already formatted)
    if (cfg->type == SWAP_PARTITION && !IsSwapFormatted(cfg->device_path)) {
        ret = system_cmd("mkswap %s", cfg->device_path);
        if (ret) return ret;
    }

    // 4. Enable swap with priority and discard
    char cmd[512];
    snprintf(cmd, sizeof(cmd), "swapon -p %d %s %s",
             cfg->priority,
             cfg->enable_discard ? "-o discard=pages" : "",
             cfg->device_path);
    return system_cmd(cmd);
}

3.3 Kernel Tuning Parameters

SMAP should configure kernel parameters at swap initialization for optimal NVMe performance:

void TuneKernelSwapParams(void) {
    // Low swappiness: let SMAP control what gets swapped, not kernel LRU
    write_sysctl("vm/swappiness", "10");

    // Disable page-cluster readahead (NVMe is fast enough for single-page I/O)
    write_sysctl("vm/page-cluster", "0");

    // Enable VMA-based readahead (better hit rate on NVMe)
    write_file("/sys/kernel/mm/swap/vma_ra_enabled", "1");
}
Important: Setting vm.swappiness=10 (not 0) ensures the kernel doesn’t aggressively swap on its own, but still allows MADV_PAGEOUT to work. With swappiness=0, the kernel refuses to swap unless under extreme memory pressure, which could cause MADV_PAGEOUT to fail silently.

3.4 Zswap Interaction

If zswap is enabled (common on openEuler), MADV_PAGEOUT pages go through zswap first:

Zswap StateMADV_PAGEOUT BehaviorRecommendation
DisabledPages go directly to NVMe swapSimplest, predictable latency
Enabled, writeback=1Pages compressed in RAM first, evicted to NVMe on pool pressureGood — free compression tier
Enabled, writeback=0Pages compressed but never reach NVMeBad — defeats the purpose
// Ensure zswap writeback is enabled if zswap is active
void ValidateZswapConfig(void) {
    if (read_file_bool("/sys/module/zswap/parameters/enabled")) {
        // Ensure writeback is on
        write_file("/sys/module/zswap/parameters/writeback_enabled", "Y");
        SMAP_LOGGER_INFO("Zswap active with writeback: pages will be compressed before NVMe");
    }
}

4. Cold Page Detection

4.1 Existing Frequency Tracking

SMAP already tracks per-page access frequency via the ActcData array. After each scan cycle, FillActcByBitmap() in manage.c populates:

// Per-page data (already exists)
actcData[nid][i].freq  // Access frequency in current scan window (0-256)
actcData[nid][i].addr  // Physical address of page

// Summary statistics (already exists)
actCount[nid].freqZero // Count of pages with freq=0 in this window
actCount[nid].freqNum  // Count of pages with freq>0

Pages with freq=0 are already identified — they are sorted to the front when ActcFreqAscFunc() sorts in ascending order. The existing CalcSwapNum4K() function at line 373 of separate_strategy.c already uses freqZero to count cold pages.

4.2 Multi-Window Cold Tracking (NEW)

A page with freq=0 in one window might be temporarily idle. To avoid swapping pages that will be accessed soon, we require pages to have freq=0 for N consecutive scan windows before becoming swap-eligible.

// NEW: Per-page cold window counter
// Stored as a parallel array to actcData[], indexed by physical address

typedef struct {
    uint64_t page_count;             // Number of tracked L2 pages
    uint64_t *phys_addrs;            // Sorted physical address array
    uint8_t *cold_windows;            // cold_windows[i] for phys_addrs[i]
    uint8_t threshold;                // Consecutive windows required (default: 5)
} ColdTracker;

// Per-process, per-L2-NUMA-node tracking
typedef struct {
    ColdTracker tracker[REMOTE_NUMA_NUM];  // One per L2 node
} ProcessColdState;

// Called after BuildAllPidData() in each ScanMigrateWork cycle
void UpdateColdWindowCounters(ProcessAttr *process) {
    for (int nid = LOCAL_NUMA_NUM; nid < MAX_NODES; nid++) {
        if (NotInAttrL2(process, nid)) continue;
        ActcData *data = process->scanAttr.actcData[nid];
        uint64_t len = process->scanAttr.actcLen[nid];
        ColdTracker *ct = &process->coldState.tracker[nid - LOCAL_NUMA_NUM];

        // Rebuild if page set changed
        if (ct->page_count != len) {
            RebuildColdTracker(ct, data, len);
        }

        // Update counters
        for (uint64_t i = 0; i < len; i++) {
            if (data[i].freq == 0) {
                if (ct->cold_windows[i] < 255)
                    ct->cold_windows[i]++;  // Saturating increment
            } else {
                ct->cold_windows[i] = 0;   // Reset on any access
            }
        }
    }
}

5. Swap-Out Path: process_madvise(MADV_PAGEOUT)

5.1 Syscall Mechanics

// Linux 5.10+ syscall
ssize_t process_madvise(
    int pidfd,                    // PID file descriptor from pidfd_open()
    const struct iovec *iovec,    // Array of {base_addr, length} ranges
    size_t vlen,                  // Number of iovec entries
    int advice,                   // MADV_PAGEOUT (21)
    unsigned int flags             // Must be 0
);

// Return: bytes successfully advised (may be less than requested)
// Requires: CAP_SYS_NICE (Linux 5.12+), swap must be active
Key semantics: MADV_PAGEOUT is a hint. The kernel will attempt to reclaim the pages, but success is not guaranteed. Pages may be skipped if they are locked, mapped by multiple processes, or if the swap device is full. The return value indicates how many bytes were processed, not how many were actually swapped.

5.2 pidfd Management

// Cache pidfd per process to avoid repeated syscalls
typedef struct {
    pid_t pid;
    int pidfd;        // -1 if not open
} PidfdCache;

// ProcessAttribute addition:
int swap_pidfd;   // Cached pidfd, -1 = not opened

static int GetOrOpenPidfd(ProcessAttr *process) {
    if (process->swap_pidfd >= 0)
        return process->swap_pidfd;

    int fd = syscall(SYS_pidfd_open, process->pid, 0);
    if (fd < 0) {
        SMAP_LOGGER_ERROR("pidfd_open(%d) failed: %s", process->pid, strerror(errno));
        return -errno;
    }
    process->swap_pidfd = fd;
    return fd;
}

// Close pidfd when process is removed from management
static void ClosePidfd(ProcessAttr *process) {
    if (process->swap_pidfd >= 0) {
        close(process->swap_pidfd);
        process->swap_pidfd = -1;
    }
}

5.3 Batching & Rate Limiting

Performance hazard: process_madvise() walks page tables, which can block for seconds on processes with large RSS. Unbounded calls can stall SMAP’s periodic migration thread. 
#define SWAP_BATCH_SIZE      64     // iovec entries per syscall
#define MAX_SWAP_PER_CYCLE   1024   // Max pages per ScanMigrateWork cycle
#define SWAP_YIELD_US        1000   // usleep between batches to avoid stalling

int DoSwapOut(ProcessAttr *process, uint64_t *swap_addrs, uint64_t nr_pages) {
    int pidfd = GetOrOpenPidfd(process);
    if (pidfd < 0) return pidfd;

    uint64_t pages_done = 0;
    uint64_t pages_remaining = MIN(nr_pages, MAX_SWAP_PER_CYCLE);
    struct iovec iov[SWAP_BATCH_SIZE];

    while (pages_done < pages_remaining) {
        int batch = MIN(SWAP_BATCH_SIZE, pages_remaining - pages_done);

        // Convert physical addresses to virtual addresses for this process
        for (int i = 0; i < batch; i++) {
            uint64_t vaddr = PhysToVirt(process, swap_addrs[pages_done + i]);
            if (vaddr == 0) continue;  // Skip unmapped pages
            iov[i].iov_base = (void *)vaddr;
            iov[i].iov_len = GetPageSize();  // 4K or 2M
        }

        // Issue syscall
        ssize_t ret = syscall(SYS_process_madvise, pidfd, iov, batch, MADV_PAGEOUT, 0);
        if (ret < 0) {
            if (errno == EAGAIN) {
                usleep(SWAP_YIELD_US);
                continue;  // Retry
            }
            SMAP_LOGGER_ERROR("process_madvise pid %d failed: %s", process->pid, strerror(errno));
            break;
        }

        pages_done += batch;

        // Yield between batches to avoid monopolizing CPU
        if (pages_done < pages_remaining)
            usleep(SWAP_YIELD_US);
    }

    // Update accounting
    UpdateSwapAccounting(process);
    return pages_done;
}

5.4 Transparent Hugepage Handling

Limitation: When MADV_PAGEOUT encounters a THP (2M hugepage), the kernel splits it into 4K pages before swapping. The hugepage is not reconstructed on swap-in — pages return as 4K. This causes performance regression for VM workloads using 2M pages.

Mitigation strategies:

  1. Prefer 4K mode for swap: Only swap pages from processes tracked in PAGETYPE_NORMAL (4K) mode
  2. Avoid swapping VM pages: Set enableSwap=false for VM_TYPE processes by default
  3. Higher cold threshold for VMs: If VM swap is enabled, use a much higher cold_window_threshold (e.g., 20 vs 5)
  4. Accept the tradeoff: For severely cold pages (freq=0 for 20+ windows), losing hugepage status is acceptable

5.5 Error Handling & Partial Success

// process_madvise returns bytes, not pages. Handle partial success:
ssize_t bytes_advised = syscall(SYS_process_madvise, pidfd, iov, vlen, MADV_PAGEOUT, 0);

if (bytes_advised < 0) {
    switch (errno) {
    case ESRCH:   // Process died
        ClosePidfd(process);
        RemoveManagedProcess(1, &process->pid);
        break;
    case EACCES:  // Permission denied (need CAP_SYS_NICE)
        SMAP_LOGGER_ERROR("Need CAP_SYS_NICE for process_madvise");
        swapPolicy.swap_enabled = false;  // Disable swap globally
        break;
    case EAGAIN:  // Temporary failure, retry next cycle
        break;
    case EINVAL:  // Bad address (page already unmapped)
        // Skip this page, continue with next batch
        break;
    case ENOMEM:  // Swap device full
        SMAP_LOGGER_WARN("Swap device full, pausing swap-out");
        break;
    }
} else if (bytes_advised < total_requested) {
    // Partial success: some iovecs processed, some failed
    uint64_t pages_done = bytes_advised / GetPageSize();
    SMAP_LOGGER_INFO("Partial swap: %lu/%lu pages", pages_done, nr_pages);
}

6. Swap-In Path

No SMAP code needed for swap-in. When a swapped page is accessed, the CPU generates a page fault. The kernel’s do_swap_page() handler automatically reads the page from NVMe swap, allocates a new page frame, and installs the PTE. This is completely transparent to SMAP.

The kernel places the faulted page on the local NUMA node of the faulting CPU. This means swap-in goes L3 → L1 directly (not back to L2), which is the desired behavior — a page that was cold enough to reach L3 but is now accessed again should return to fast memory.

// Kernel swap-in path (no SMAP involvement):
//
// 1. CPU accesses swapped page → page fault
// 2. do_swap_page() reads swap entry from PTE
// 3. swap_readpage() → bio → NVMe driver → page data
// 4. Allocate new page on faulting CPU's local NUMA node
// 5. Install PTE, page now on L1
//
// SMAP sees the page on L1 in the next scan cycle via actcData[l1Node]
// If the page stays hot, SMAP keeps it on L1
// If the page goes cold again, normal DEMOTE to L2, then eventually SWAP_OUT again

SMAP should update its accounting when it detects a process’s VmSwap decreased between cycles, indicating swap-in activity:

void DetectSwapIn(ProcessAttr *process) {
    uint64_t current_swap = ReadVmSwap(process->pid);
    if (current_swap < process->swapAccounting.last_vm_swap) {
        uint64_t swapped_in = process->swapAccounting.last_vm_swap - current_swap;
        process->swapAccounting.total_swap_in_kb += swapped_in;
        SMAP_LOGGER_INFO("pid %d swap-in detected: %lu KB", process->pid, swapped_in);
    }
    process->swapAccounting.last_vm_swap = current_swap;
}

7. Swap Accounting & Monitoring

typedef struct {
    uint64_t current_swap_kb;       // Current VmSwap from /proc/pid/status
    uint64_t last_vm_swap;          // Previous cycle's VmSwap (for delta detection)
    uint64_t total_swap_out_kb;     // Cumulative pages swapped out
    uint64_t total_swap_in_kb;      // Cumulative pages swapped in (detected via delta)
    uint64_t swap_out_failures;     // Failed process_madvise calls
    uint64_t max_swap_kb;           // Per-process swap limit (0=unlimited)
} SwapAccounting;

// Read VmSwap from /proc/[pid]/status
static uint64_t ReadVmSwap(pid_t pid) {
    char path[64];
    snprintf(path, sizeof(path), "/proc/%d/status", pid);
    FILE *f = fopen(path, "r");
    if (!f) return 0;

    char line[256];
    uint64_t swap_kb = 0;
    while (fgets(line, sizeof(line), f)) {
        if (strncmp(line, "VmSwap:", 7) == 0) {
            sscanf(line + 7, "%lu", &swap_kb);
            break;
        }
    }
    fclose(f);
    return swap_kb;
}

// Global swap device monitoring
typedef struct {
    uint64_t total_kb;          // Total swap capacity
    uint64_t used_kb;           // Currently used
    uint64_t free_kb;           // Available
    double usage_ratio;         // used / total
} SwapDeviceStatus;

8.2 Integration with SeparateStrategy

The swap-out logic is not inside SeparateStrategy(). It runs as a separate phase after the existing migration to avoid coupling. The integration point is in ScanMigrateWork():

// Modified ScanMigrateWork() in migration.c
int ScanMigrateWork(ThreadCtx *ctx) {
    struct ProcessManager *manager = ctx->processManager;

    // === EXISTING: Phase 1 — L1 ↔ L2 migration ===
    CheckAndRemoveInvalidProcess();
    BuildAllPidData();
    CleanStrategyAttribute(manager);
    PerformMigrationPreparation(manager);

    struct MigrateMsg mMsg;
    InitMigrateMsg(&mMsg, manager);
    uint64_t migratePage = 0;

    ProcessAttr *current;
    for (current = manager->processes; current; current = current->next) {
        BuildMigrationMsg(current, &mMsg, &migratePage);
    }
    if (mMsg.cnt > 0) {
        DoMigration(&mMsg, manager);
        UpdateMigResult(&mMsg, manager);
    }

    // === NEW: Phase 2 — L2 → L3 swap-out ===
    if (manager->swapPolicy.swap_enabled && IsSwapDeviceActive()) {
        for (current = manager->processes; current; current = current->next) {
            UpdateColdWindowCounters(current);

            if (!current->enableSwap) continue;
            if (current->type == VM_TYPE && !manager->swapPolicy.allow_vm_swap) continue;

            uint64_t nr_candidates;
            uint64_t *swap_addrs = SelectSwapCandidates(current, &nr_candidates);
            if (nr_candidates > 0) {
                int swapped = DoSwapOut(current, swap_addrs, nr_candidates);
                SMAP_LOGGER_INFO("pid %d swapped %d/%lu pages to NVMe",
                                 current->pid, swapped, nr_candidates);
            }
            free(swap_addrs);
        }
    }

    return 0;
}

  • Implement PhysToVirt() using /proc/pid/pagemap (leverage existing walk_pid_pagemap logic)
  • Add Phase 2 swap-out loop to ScanMigrateWork()
  • Add DetectSwapIn() for swap-in accounting via VmSwap delta
Phase 4: Config & API (2 days)
  • Implement swap_config.c: persist/recover from /dev/shm/smap_swap_config
  • Add SDK API functions: ubturbo_smap_swap_config_set(), ubturbo_smap_swap_stats_query()
  • Wire through IPC codec + server handler
Phase 5: Testing & Tuning (3–5 days)
  • Unit test: cold tracker with synthetic frequency data
  • Integration test: end-to-end swap-out with real NVMe device
  • Stress test: 100+ processes, monitor swap throughput and latency
  • Benchmark: compare with manual swapoff/swapon cycles
  • Tune defaults: cold_window_threshold, batch_size, watermark_ratio
  • Test zswap interaction scenarios

Total estimated effort: ~2–3 weeks for one developer familiar with the SMAP codebase.
