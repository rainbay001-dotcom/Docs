1. Design Overview & Rationale

Option B v1 used process_madvise(MADV_PAGEOUT) — a pure user-space approach that required zero kernel changes. While conceptually simple, a critical limitation was discovered: process_madvise operates at VMA granularity (contiguous virtual address ranges), not individual physical pages. SMAP’s cold detection, however, produces per-page bitmap indices via actcData[].addr, which are not virtual addresses.

Problem discovered: actcData[nid][i].addr is a bitmap index (0, 1, 2, ...), not a physical or virtual address. The kernel’s convert_pos_to_paddr_sorted() does the index → physical address conversion internally. A pure user-space approach cannot perform this conversion without kernel cooperation.

Option B v2 solves this with a single new ioctl (SMAP_MIG_SWAP_OUT) in the existing kernel module. This ioctl reuses 90% of the migration infrastructure — specifically build_migrate_list() for index-to-PA conversion — then calls fp_reclaim_pages() instead of fp_migrate_pages(). The result: per-page swap control with minimal kernel changes.

Key insight: The existing migration path already solves the hard problem (bitmap index → physical address → folio). We reuse that entire pipeline, substituting only the final step: reclaim_pages() instead of migrate_pages().

4. The actcData.addr Problem

Understanding why Option B v1 failed requires understanding how SMAP tracks page access frequency. The kernel module maintains a bitmap (paddr_bm[]) where each bit represents a physical page. When user-space queries access data, the kernel returns:

struct ActcData {
    unsigned long addr;   // NOT a physical address — this is a bitmap INDEX
    unsigned int  freq;   // access frequency count for this scan window
};

// In manage.c:461 (during kernel scan):
actcData[nid][actcLen].addr = actcLen;   // sequential index 0, 1, 2, ...
actcData[nid][actcLen].freq = freq;
actcLen++;

The .addr field is a position counter, not an address. The kernel’s convert_pos_to_paddr_sorted() walks the bitmap to convert these positions to actual physical addresses. This function is only available in kernel space.

Why this matters: Option B v1 attempted to use process_madvise() which requires virtual addresses and operates per-VMA. But SMAP’s access tracking produces bitmap indices that can only be converted to physical addresses inside the kernel. The v2 ioctl approach keeps everything in the kernel’s address space, reusing the exact same conversion code that migration already uses.

5. Kernel Changes (Minimal)

5.1 New kprobe: reclaim_pages()

The reclaim_pages() function (available since Linux 5.16) forces page reclaim for a given list of folios, writing them to the configured swap device. We resolve it at module init time using the same kallsyms_lookup_name pattern already used for fp_migrate_pages.

/* smap_migrate_wrapper.h */
extern unsigned long (*fp_reclaim_pages)(struct list_head *folio_list);

/* smap_migrate_wrapper.c — in smap_resolve_symbols() */
fp_reclaim_pages = (void *)kallsyms_lookup_name("reclaim_pages");
if (!fp_reclaim_pages) {
    pr_warn("smap: reclaim_pages not found, swap-out disabled\n");
    /* Module continues to work — only swap-out is unavailable */
}
Graceful fallback: If reclaim_pages is not found (older kernels), the module loads normally but the SMAP_MIG_SWAP_OUT ioctl returns -ENOSYS. All existing functionality (L1/L2 migration) is unaffected.

5.2 New ioctl: SMAP_MIG_SWAP_OUT

5.2.1 ioctl Definition

/* mig_init.h */
#define SMAP_MIG_MAGIC     0xB9
#define SMAP_MIG_MIGRATE   _IOWR(SMAP_MIG_MAGIC, 1, MigrateMsg)   /* existing */
#define SMAP_MIG_SWAP_OUT  _IOWR(SMAP_MIG_MAGIC, 4, MigrateMsg)   /* NEW */

5.2.2 do_swap_out() — Core Swap Logic

/*
 * do_swap_out — Convert physical addresses to folios, isolate, and reclaim.
 * Reuses the same mig_list produced by build_migrate_list().
 * Each entry in mig_list has .paddr set by convert_pos_to_paddr_sorted().
 */
static int do_swap_out(struct list_head *mig_list, MigrateMsg *mMsg)
{
    struct mig_entry *entry;
    struct page *page;
    struct folio *folio;
    LIST_HEAD(folio_list);
    int nr_isolated = 0, nr_reclaimed = 0;
    unsigned long nr_total = 0;

    if (!fp_reclaim_pages)
        return -ENOSYS;

    /* Phase 1: Convert PAs to folios and isolate */
    list_for_each_entry(entry, mig_list, list) {
        unsigned long pfn;
        nr_total++;

        if (!entry->paddr)
            continue;

        pfn = entry->paddr >> PAGE_SHIFT;
        if (!pfn_valid(pfn))
            continue;

        page = pfn_to_page(pfn);
        folio = page_folio(page);

        if (!folio_try_get(folio))
            continue;

        if (folio_test_lru(folio) && !folio_test_unevictable(folio)) {
            if (fp_isolate_folio_to_list(folio, &folio_list) == 0)
                nr_isolated++;
            else
                folio_put(folio);
        } else {
            folio_put(folio);
        }
    }

    /* Phase 2: Reclaim isolated folios to swap */
    if (nr_isolated > 0)
        nr_reclaimed = fp_reclaim_pages(&folio_list);

    /* Report results */
    mMsg->result.nr_total     = nr_total;
    mMsg->result.nr_succeeded = nr_reclaimed;
    mMsg->result.nr_failed    = nr_isolated - nr_reclaimed;

    pr_debug("smap: swap_out total=%lu isolated=%d reclaimed=%d\n",
             nr_total, nr_isolated, nr_reclaimed);

    return 0;
}

5.2.3 __ioctl_swap_out() — ioctl Entry Point

/*
 * __ioctl_swap_out — Entry point for SMAP_MIG_SWAP_OUT ioctl.
 * Same structure as __ioctl_migrate(): copy, validate, build, act, return.
 */
static long __ioctl_swap_out(unsigned long arg)
{
    MigrateMsg mMsg;
    struct list_head mig_list;
    int ret;

    /* 1. Copy message from user-space */
    if (copy_from_user(&mMsg, (void __user *)arg, sizeof(MigrateMsg)))
        return -EFAULT;

    /* 2. Validate */
    if (!is_migrate_msg_valid(&mMsg))
        return -EINVAL;

    /* 3. Build list: bitmap indices → physical addresses */
    INIT_LIST_HEAD(&mig_list);
    ret = build_migrate_list(&mMsg, &mig_list);
    if (ret) {
        free_migrate_list(&mig_list);
        return ret;
    }

    /* 4. convert_pos_to_paddr_sorted is called inside build_migrate_list */

    /* 5. Swap out: isolate folios and reclaim */
    ret = do_swap_out(&mig_list, &mMsg);

    /* 6. Cleanup */
    free_migrate_list(&mig_list);

    /* 7. Copy results back to user-space */
    if (copy_to_user((void __user *)arg, &mMsg, sizeof(MigrateMsg)))
        return -EFAULT;

    return ret;
}

5.2.4 ioctl Dispatch (switch case addition)

/* mig_init.c — in smap_mig_ioctl() */
case SMAP_MIG_SWAP_OUT:
    ret = __ioctl_swap_out(arg);
    break;

6. User-Space Changes

6.1 Cold Tracker (strategy/cold_tracker.c) — Same as v1

The cold tracker maintains a per-page cold_windows counter. Each scan cycle:

  • If actcData[nid][i].freq == 0: increment cold_windows[nid][i] (saturate at 255)
  • If actcData[nid][i].freq > 0: reset cold_windows[nid][i] = 0
void UpdateColdWindowCounters(ProcessInfo *proc, int nid)
{
    for (int i = 0; i < proc->actcLen[nid]; i++) {
        if (proc->actcData[nid][i].freq == 0) {
            if (proc->cold_windows[nid][i] < 255)
                proc->cold_windows[nid][i]++;
        } else {
            proc->cold_windows[nid][i] = 0;
        }
    }
}

6.2 Swap Device Manager (manage/swap_device.c) — Same as v1

Manages the NVMe swap device lifecycle: checks if already active via /proc/swaps, calls swapon() with specified priority, monitors device health and free capacity.

int SwapDeviceInit(const char *device_path, int priority)
{
    /* Check if already active */
    if (IsSwapActive(device_path))
        return 0;

    /* Activate via swapon syscall */
    int ret = swapon(device_path, SWAP_FLAG_PREFER | (priority & SWAP_FLAG_PRIO_MASK));
    if (ret < 0) {
        SMAP_LOG_ERR("swapon(%s) failed: %s", device_path, strerror(errno));
        return -1;
    }

    SMAP_LOG_INFO("NVMe swap activated: %s (priority %d)", device_path, priority);
    return 0;
}

6.3 Swap Accounting (manage/swap_account.c) — Same as v1

Tracks per-process swap usage by reading /proc/[pid]/status VmSwap field. Detects swap-in events (VmSwap decrease) without any kernel modification.

void UpdateSwapAccounting(ProcessInfo *proc)
{
    unsigned long current_swap = ReadVmSwap(proc->pid);
    long delta = (long)current_swap - (long)proc->last_vmswap;

    if (delta > 0)
        proc->swap_out_bytes += delta;     /* pages swapped out */
    else if (delta < 0)
        proc->swap_in_bytes += (-delta);   /* pages swapped in via fault */

    proc->last_vmswap = current_swap;
}

6.4 DoSwapOut (strategy/swap_out.c) — CHANGED from v1

Key change from v1: No more process_madvise, no pidfd, no PA-to-VA conversion. Instead, we build a MigrateMsg with bitmap indices (grouped by L2 NUMA node) and call the existing migration ioctl interface with the new SMAP_MIG_SWAP_OUT command. The kernel handles everything from index conversion to reclaim. 
int DoSwapOut(SmapManager *manager, ProcessInfo *proc,
              int *candidates, int nr_candidates, int src_nid)
{
    MigrateMsg mMsg;
    int ret;

    if (nr_candidates == 0)
        return 0;

    /* Build MigrateMsg — same format as existing migration */
    memset(&mMsg, 0, sizeof(mMsg));
    mMsg.pid       = proc->pid;
    mMsg.src_nid   = src_nid;       /* L2 NUMA node */
    mMsg.dst_nid   = -1;            /* unused for swap-out */
    mMsg.nr_pages  = nr_candidates;

    /* Fill page indices — these are bitmap positions, not addresses */
    for (int i = 0; i < nr_candidates && i < MAX_MIGRATE_PAGES; i++)
        mMsg.pages[i] = candidates[i];

    /* Call the NEW ioctl — uses the same fd as migration */
    ret = ioctl(manager->fds.migrate, SMAP_MIG_SWAP_OUT, &mMsg);
    if (ret < 0) {
        SMAP_LOG_ERR("SMAP_MIG_SWAP_OUT failed: %s", strerror(errno));
        return -1;
    }

    SMAP_LOG_DEBUG("swap_out: total=%lu ok=%lu fail=%lu",
        mMsg.result.nr_total, mMsg.result.nr_succeeded, mMsg.result.nr_failed);

    return mMsg.result.nr_succeeded;
}
Simplicity: DoSwapOut() is structurally identical to the existing DoMigrate() call. Same message format, same fd, same copy semantics. The only difference is the ioctl command number.

6.5 SDK API (smap_interface.c)

Two new public API functions for SDK integration:

/**
 * Configure and activate NVMe swap for SMAP.
 * @device_path  Path to NVMe partition (e.g., "/dev/nvme0n1p3")
 * @priority     Swap priority (higher = preferred, default 10)
 * @enable       1 to enable, 0 to disable
 */
int ubturbo_smap_swap_config(const char *device_path, int priority, int enable)
{
    if (enable) {
        int ret = SwapDeviceInit(device_path, priority);
        if (ret) return ret;

        TuneKernelSwapParams();    /* swappiness, page-cluster, etc. */
        ValidateZswapConfig();     /* ensure writeback is enabled */

        g_swap_enabled = 1;
    } else {
        g_swap_enabled = 0;
    }
    return 0;
}

/**
 * Set the cold window threshold for swap eligibility.
 * Pages must be cold for this many consecutive scan windows before swap-out.
 */
int ubturbo_smap_swap_set_threshold(int threshold)
{
    if (threshold < 1 || threshold > 255)
        return -EINVAL;
    g_cold_window_threshold = threshold;
    return 0;
}

6.6 Integration in ScanMigrateWork (strategy/migration.c)

Swap-out runs as Phase 2 after the existing L1/L2 migration (Phase 1). It executes within the same scan cycle, using the same access data that Phase 1 already loaded:

void ScanMigrateWork(SmapManager *manager)
{
    /* Phase 1: Existing L1 ↔ L2 migration (unchanged) */
    for (int i = 0; i < manager->nr_procs; i++) {
        ProcessInfo *proc = &manager->procs[i];
        DoMigration(manager, proc);  /* existing code */
    }

    /* Phase 2: NVMe swap-out for cold L2 pages (NEW) */
    if (!g_swap_enabled)
        return;

    for (int i = 0; i < manager->nr_procs; i++) {
        ProcessInfo *proc = &manager->procs[i];

        /* Iterate L2 NUMA nodes */
        for (int nid = 0; nid < manager->nr_l2_nodes; nid++) {
            int l2_nid = manager->l2_nodes[nid];

            /* Step 1: Update cold counters from this cycle’s freq data */
            UpdateColdWindowCounters(proc, l2_nid);

            /* Step 2: Select pages exceeding cold threshold */
            int candidates[MAX_SWAP_PER_CYCLE];
            int nr = SelectSwapCandidates(proc, l2_nid,
                        candidates, MAX_SWAP_PER_CYCLE);

            /* Step 3: Issue swap-out ioctl */
            if (nr > 0)
                DoSwapOut(manager, proc, candidates, nr, l2_nid);

            /* Step 4: Update accounting */
            UpdateSwapAccounting(proc);
        }
    }
}

7. Swap-In Path

Swap-in is handled entirely by the kernel with zero SMAP involvement:

  1. Application accesses a swapped-out page
  2. CPU generates a page fault (PTE contains swap entry, not page mapping)
  3. Kernel’s do_swap_page() handles the fault:
    • Allocates a new page on the faulting CPU’s local NUMA node (L1 direct)
    • Calls swap_readpage() to read the page data from NVMe
    • Updates PTE to point to the new physical page
    • Frees the swap slot
  4. Application resumes — page is now in L1 local DRAM
Optimal placement: Swap-in always places pages on L1 (local DRAM) because the kernel allocates on the faulting CPU’s NUMA node. A cold page that was on L2 before swap-out gets promoted directly to L1 on swap-in — no intermediate L2 step.

SMAP detects swap-in events passively by monitoring VmSwap decrease in UpdateSwapAccounting(). This is purely informational — no action is needed because the kernel has already placed the page optimally.

13. References

  1. reclaim_pages() — Added in Linux 5.16 (mm/vmscan.c). Forces reclaim of a list of folios, writing dirty/anonymous pages to swap. Used by MADV_PAGEOUT internally and by memory cgroup reclaim.
  2. process_madvise(2) — Linux 5.10+. Allows a process to advise the kernel about another process’s memory. MADV_PAGEOUT reclaims pages but operates at VMA granularity, not per-page.
  3. SMAP Migration Infrastructure — Existing kernel module code: build_migrate_list(), convert_pos_to_paddr_sorted(), fp_migrate_pages(), fp_isolate_folio_to_list(). The v2 design reuses all of these except the final fp_migrate_pages() call.
  4. kallsyms_lookup_name() — Kernel function to resolve unexported symbols at runtime. Already used by SMAP for fp_migrate_pages, fp_isolate_folio_to_list, and other kprobed functions.
  5. Option B v1 Design — smap_nvme_swap_option_b.html — Original pure user-space design using process_madvise(MADV_PAGEOUT).
  6. Option A Design — smap_nvme_swap_option_a.html — Full custom kernel design with submit_bio(), custom PTE handling, 2M swap slots.
  7. Linux Swap Subsystem — mm/swap_state.c, mm/swapfile.c, mm/page_io.c. Handles swap entry allocation, page I/O, and PTE management for the standard swap path.
  8. folio API — Linux 5.16+ page cache and LRU management. folio_try_get(), folio_test_lru(), folio_test_unevictable(), folio_put().

Option B v2 — Per-page swap with ~110 lines of kernel code. The pragmatic middle ground.
